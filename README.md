# LoRA Dataset Preparer

一个用于本地整理 LoRA / 文生图训练数据的小工具，支持图片导入、预览裁切、批量重命名、AI 打标与导出。

本项目当前使用的是**外接 LLM API**进行打标，API 配置保存在**浏览器本地存储**中，不会自动写入仓库文件。

## 功能简介

- 图片批量上传
- 双击图片进入大图预览
- 按预设尺寸裁切图片
- 在预览页拖动白色裁切框，自定义每张图片的裁切区域
- 批量重命名
- 调用外部 LLM API 进行标签 / Caption 生成
- 中英文界面切换
- 导出训练数据

## 运行环境

- Node.js 18 及以上
- Windows / macOS / Linux

## 安装与启动

## 新电脑第一次使用

如果你是在一台新电脑上第一次运行本项目，请先确认：

- 已安装 `Node.js 18` 或更高版本
- 电脑可以正常联网
- 没有被防火墙或安全软件拦截本地 `localhost` 服务

安装好 Node.js 后，再运行项目。

### Windows 用户：推荐直接双击 `start.bat`

项目根目录下提供了 `start.bat`，适合普通用户直接启动。

双击后它会自动执行以下操作：

- 检查当前电脑是否已安装 Node.js
- 如果没有安装依赖，则自动执行 `npm install`
- 自动启动本地开发服务
- 自动打开浏览器并访问 `http://localhost:3000`

如果浏览器没有自动打开，也可以手动访问：

`http://localhost:3000`

### 手动启动方式

如果你不想用 `start.bat`，也可以在项目根目录手动执行：

```bash
npm install
npm run dev
```

启动成功后，终端通常会显示：

```bash
Server running on http://localhost:3000
```

## 使用说明

## 一、启动后先做什么

第一次打开网页后，建议先点击右上角的设置按钮，填写你的模型接口配置。

你需要准备：

- `Base URL`
- `API Key`
- `Model`

如果你不确定怎么填，可以先在设置里选择预设接口，例如：

- 阿里千问（兼容模式）
- 豆包 Ark（兼容模式）
- SiliconFlow
- Moonshot / Kimi
- 自定义

选择预设后，系统会自动帮你填入常见的接口地址和模型名，你再补上自己的 `API Key` 即可。

## 二、上传图片

- 点击右上角的“上传图片”按钮
- 支持一次选择多张图片
- 上传后图片会显示在图库中

## 三、预览和裁切

- 在图库中双击任意图片，可以进入大图预览
- 右侧可以选择裁切预设，或手动输入自定义宽高
- 白色裁切框支持拖动
- 每张图片的裁切位置会分别保存
- 点击“应用裁切”后，会按每张图自己的裁切区域执行处理

## 四、批量重命名

- 在右侧重命名区域输入文件名前缀
- 可对当前选中的图片应用
- 也可一次应用到全部图片

## 五、AI 打标

- 确认右上角设置中已经填写好 API 配置
- 切换到打标页面
- 系统会根据图片内容生成：
  - 标签列表（Tags）
  - 自然语言描述（Caption）

当前打标提示词已经偏向训练数据整理场景：

- 标签会相对丰富一些
- 但会尽量避免脑补
- 更方便你后续手动筛选和二次处理

## 六、导出数据

- 点击右上角“导出 ZIP”
- 导出当前处理后的图片和相关数据

## 七、语言切换

- 点击顶部的语言按钮，可以在中文和英文之间切换界面文字

## API 配置说明

本项目**不内置任何真实 API Key**。

你需要在页面中自行填写：

- `Base URL`
- `API Key`
- `Model`

这些配置会保存在浏览器 `localStorage` 中，仅保存在你当前使用的浏览器环境里。

这意味着：

- 关闭网页后，再次打开，配置还会保留
- 这些内容默认不会自动写入代码文件
- 这些内容默认不会自动进入 Git 仓库

但也请注意：

- `localStorage` 属于本机浏览器本地存储，不是加密保险箱
- 如果多人共用同一台电脑或同一个浏览器配置，请谨慎保存你的 API Key

## 常见问题

### 1. 双击 `start.bat` 后打不开网页

请优先检查：

- 是否已安装 Node.js 18+
- 首次运行时是否正在安装依赖
- 是否有安全软件拦截本地服务
- `3000` 端口是否被其他程序占用

如果浏览器显示无法访问 `localhost:3000`，请查看启动时弹出的命令行窗口里是否有报错。

### 2. 为什么关闭网页再打开，API Key 还在

这是正常现象。

因为当前配置保存在浏览器本地的 `localStorage` 中，所以刷新页面或重新打开网页后仍然会保留。

### 3. 国内模型能不能用

可以，但优先建议使用 **OpenAI Chat Completions 兼容格式** 的接口。

如果平台提供的是兼容模式，通常可以直接接入。
如果平台提供的是原生 SDK 或原生 API，而不是兼容接口，则建议使用“自定义”并自行确认参数格式是否兼容。

## 开源与安全说明

在公开到 GitHub 前，建议你确认以下几点：

- 不要提交 `.env`、`.env.local` 等环境变量文件
- 不要提交包含真实 API Key 的截图、测试数据或日志
- 不要把浏览器导出的本地存储数据公开
- 本项目已经移除了前端构建中的环境变量注入配置，避免将本地密钥打进前端包中
- 如果后续要部署到公网，建议给服务端代理接口增加目标地址白名单，避免被滥用为通用代理
